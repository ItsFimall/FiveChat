## 🛠️ 优化建议

### 开发环境
1. **使用 Turbo 模式**
   ```bash
   npm run dev  # 已默认启用 --turbo
   ```

2. **启用性能监控**
   - 在开发时启用 PerformanceMonitor 组件
   - 监控消息数量和渲染时间

### 生产环境
1. **数据库优化**
   - 添加适当的索引
   - 定期清理旧消息
   - 考虑使用 Redis 缓存

2. **CDN 和缓存**
   - 使用 CDN 加速静态资源
   - 启用浏览器缓存
   - 配置适当的 Cache-Control 头

3. **服务器配置**
   - 启用 Gzip/Brotli 压缩
   - 配置适当的内存限制
   - 使用负载均衡

## 📈 性能指标

### 目标指标
- **首次内容绘制 (FCP)**: < 1.5s
- **最大内容绘制 (LCP)**: < 2.5s
- **首次输入延迟 (FID)**: < 100ms
- **累积布局偏移 (CLS)**: < 0.1

### 消息列表性能
- **50 条消息以下**: 普通渲染
- **50-200 条消息**: 虚拟化渲染
- **200+ 条消息**: 分页加载 + 虚拟化

## 🔧 故障排除

### 常见性能问题

1. **消息加载慢**
   - 检查数据库查询是否有索引
   - 确认分页参数是否合理
   - 检查网络连接状态

2. **滚动卡顿**
   - 确认虚拟化是否正常工作
   - 检查消息组件是否正确使用 memo
   - 减少不必要的重新渲染

3. **内存使用过高**
   - 清理不必要的缓存
   - 检查是否有内存泄漏
   - 限制同时加载的消息数量
